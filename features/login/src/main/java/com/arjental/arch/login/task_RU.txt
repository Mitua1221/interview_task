1. Проверить, был ли передан логин пользователя на экран в args
- если логин был передан, вставить логин пользователя в поле логина
Success:
- Если пользователь существует на бекенде, загрузить картинку и загрузить пароль пользователя с бекенда
Error:
- Если пользователя не существует на бекенде, то показывать ошибку, что его не существует

2. Если пользователь изменяет логин, необходимо перезагружать картинку и пароль пользователя

3. По нажатию на кнопку входа проверять совпадение введенного пароля и пароля с бекенда
Success:
- Если пароли совпадают, то загружать токен и сохранять его в локальное защищенное хранилище
Error:
- Если пароли не совпадают, то показывать ошибку с бекенда в поле с ошибкой

- Локаль пользователя
